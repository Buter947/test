These codes are called **HTML Entities**. They are a fundamental part of how web pages work. This tool will help you both encode (create) and decode (revert) these entities, and includes a searchable reference to find the exact code you need.

Why Do HTML Entities Exist?

HTML entities serve two main purposes:

  • To Display Reserved Characters: Characters like < and > are reserved by HTML to define tags. If you want to show these characters as literal text, you must "encode" them. For example, to display the text

    on your page, you must write &lt;p&gt; in your HTML code.

  • To Display Special Characters: To display symbols that aren't on a standard keyboard (like the Copyright symbol ©, a registered trademark ®, or a currency symbol like the Euro €), you use their specific entity codes (&copy;, &reg;, &euro;).

Frequently Asked Questions (FAQ)

Q1: What's the difference between Named, Numeric, and Hex entities?

A: They all achieve the same result. Named entities are readable (e.g., &copy;). Numeric entities use the character's decimal code (e.g., &#169;). Hexadecimal entities use the character's hex code (e.g., &#xA9;). While named entities are easier to remember, numeric/hex entities are universally recognized for all characters, even those without names.

Q2: My apostrophe ' became ’. Why?

A: You're seeing the difference between a "straight quote" (') and a "smart quote" or "typographer's quote" (’). Many text editors automatically convert your straight quotes to smart quotes (‘ and ’). Our tool includes these common smart quotes in its database to help you encode and decode them properly.

Q3: Why did you use ' (or ') for the apostrophe?

A: While ' is valid in HTML5, the numeric code ' is more universally compatible, especially with older XML and XHTML standards. This tool's database uses ' but the decoder will recognize all forms.

Q4: Why does my text have &nbsp; everywhere?

A: &nbsp; is a Non-Breaking Space. It's an HTML entity used to create a space that a browser is not allowed to break into a new line. They are often inserted by web editors (like WordPress!) or when copying text from web pages. Use our decoder to convert them back to standard spaces if you don't need them.
